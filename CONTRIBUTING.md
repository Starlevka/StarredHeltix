# Contributing to StarredHeltix

Thank you for your interest in contributing to StarredHeltix! This document provides guidelines and information to help you contribute effectively.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a new branch for your feature or bug fix
4. Make your changes
5. Test your changes thoroughly
6. Commit your changes with a descriptive commit message
7. Push to your fork
8. Submit a pull request

## Development Setup

### Prerequisites
- JDK 21
- IntelliJ IDEA or another Java IDE
- Minecraft Development plugin for your IDE
- Fabric Loom

### Building
```bash
# Build the mod
./gradlew build

# Run the client
./gradlew runClient

# Run the server
./gradlew runServer
```

## Code Style

We follow standard Java conventions with some specific guidelines:

1. Use 4 spaces for indentation (no tabs)
2. Opening braces on the same line as the statement
3. Keep lines under 120 characters when possible
4. Use meaningful variable and method names
5. Comment complex logic
6. Follow existing code patterns in the project

## Pull Request Guidelines

1. Ensure your PR addresses a specific issue or adds clear value
2. Include a detailed description of the changes
3. Keep PRs focused on a single feature or bug fix
4. Ensure all tests pass
5. Add new tests if applicable
6. Update documentation as needed
7. Follow the code style guidelines

## Reporting Issues

When reporting issues, please include:

1. A clear and descriptive title
2. Detailed steps to reproduce the issue
3. Expected vs actual behavior
4. Screenshots if applicable
5. Minecraft and mod version information
6. Any relevant log files

## Feature Requests

We welcome feature requests! Please:

1. Check if the feature has already been requested
2. Clearly describe the proposed feature
3. Explain the use case and benefits
4. Consider implementation complexity

## Code of Conduct

Be respectful and constructive in all interactions. Harassment, discrimination, or any other inappropriate behavior will not be tolerated.

## Questions?

If you have any questions about contributing, feel free to ask in the issues or discussions sections of the repository.
